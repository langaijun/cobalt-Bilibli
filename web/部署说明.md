# B站下载器 / Cobalt 部署说明（详细版）

## 一、先搞清楚：Cobalt 是「前端 + 后端」两部分

| 部分 | 是什么 | 部署方式 |
|------|--------|----------|
| **前端 (web)** | 你在浏览器里看到的「B站下载器」页面：输入框、下载按钮、格式选择等。纯静态 HTML/JS/CSS。 | **Vercel、Netlify、Nginx 等都可以**，本质就是传静态文件。 |
| **后端 (API)** | 真正干活的：解析 B 站链接、拉取视频/音频、转码、生成下载地址等。一个 **Node 常驻进程**。 | **必须跑在一台「服务器」上**：Docker 或直接 Node，不能放在「只做静态」的 Vercel 上。 |

所以：

- **Vercel 只能部署「前端」**，这是对的；  
- **「有 Cobalt」** = 页面要能连上一个 **Cobalt API**。  
  - 要么用别人/官方提供的 API 地址（你只部署前端），  
  - 要么自己用 **Docker 或 Node 把 API 跑起来**（再让前端填你这个 API 地址）。

**不需要**把「页面」塞进 Docker；Docker 只用来跑 **API 服务**。

---

## 二、两种常见部署方式

### 方式 A：只部署前端，用「别人的」API（最快）

- **前端**：部署到 Vercel（或任意静态托管）。  
- **API**：不自己搭，用官方或别人公开的 API 地址，例如 `https://api.cobalt.tools`（若有限制或不可用就换别的）。  
- 在前端环境变量里把 **WEB_DEFAULT_API** 设成这个地址即可。

**适合**：先上线页面、体验流程，或流量不大、不想维护服务器。

---

### 方式 B：前端 + 自建 API（完整、可控）

- **前端**：同样部署到 Vercel（或静态托管）。  
- **API**：在自己服务器（VPS/云主机）上用 **Docker** 跑 Cobalt 官方镜像，或在本机/服务器用 `pnpm start` 跑 `api` 目录。  
- 前端 **WEB_DEFAULT_API** 填你自己 API 的域名，例如 `https://api.你的域名.com`。

**适合**：要稳定、可控、或国内访问，愿意维护一台服务器。

下面分别写具体步骤。

---

## 三、前端部署到 Vercel（两种方式都会用到）

1. 代码推到 Git（GitHub / Gitee 等）。
2. 打开 [vercel.com](https://vercel.com) → Import 你的仓库。
3. 在 Vercel 里：
   - **Root Directory** 选 **`web`**（不要选整个 monorepo 根目录）。
   - **Build Command**：`pnpm run build`（如需要可写成 `pnpm install && pnpm run build`）。
   - **Output Directory**：`build`。
4. **Environment Variables** 里加一条：
   - 名字：`WEB_DEFAULT_API`  
   - 值：
     - 方式 A：`https://api.cobalt.tools`（或别人给的 API 地址）。  
     - 方式 B：`https://你的API域名`（下面会说到怎么得到这个域名）。
5. 部署完成后，会得到一个 `xxx.vercel.app` 的地址，这就是你的「B站下载器」页面。

项目里的 `web/vercel.json` 已经配好了 COOP/COEP 头，Vercel 会自动用上，无需再改。

---

## 四、自建 API（方式 B）：用 Docker（推荐）

API 是 Node 服务，要 **常驻运行**，放不到 Vercel 上，只能放在「能跑 Docker 或 Node 的服务器」上。

### 4.1 用官方 Cobalt 镜像（最简单）

你本地的 `docs/examples/docker-compose.example.yml` 就是为这个准备的。在**有 Docker 的服务器**上：

1. 新建一个目录，例如 `cobalt`，进入：
   ```bash
   mkdir cobalt && cd cobalt
   ```

2. 创建 `docker-compose.yml`，内容示例（**把 API_URL 改成你实际对外访问的 API 地址**）：
   ```yaml
   services:
     cobalt:
       image: ghcr.io/imputnet/cobalt:11
       init: true
       read_only: true
       restart: unless-stopped
       container_name: cobalt
       ports:
         - "9000:9000/tcp"
       environment:
         # 必须改成：你最终给前端用的 API 地址，例如 https://api.你的域名.com
         API_URL: "https://api.你的域名.com/"
   ```

3. 启动：
   ```bash
   docker compose up -d
   ```
   此时 API 在本机 `9000` 端口跑。

4. 若服务器有公网 IP，用 **Nginx（或 Caddy）做反向代理**，把 `https://api.你的域名.com` 指到 `http://127.0.0.1:9000`，并配好 HTTPS（Let’s Encrypt 等）。这样：
   - 前端里的 **WEB_DEFAULT_API** 就填：`https://api.你的域名.com`。

官方文档：项目里的 `docs/run-an-instance.md`；保护实例（防滥用）：`docs/protect-an-instance.md`。

### 4.2 不用 Docker，直接跑 Node（适合开发或已有 Node 环境）

在**有 Node 18+ 和 pnpm 的机器**上：

```bash
cd d:\1\cobalt\api
pnpm install
# 新建 .env，里面至少写：API_URL=http://localhost:9000/ 或你对外域名
pnpm start
```

同样需要 Nginx 把域名反代到 `http://127.0.0.1:9000`，前端 **WEB_DEFAULT_API** 填这个域名。

## 五、总结对照表

| 你想怎么做 | 前端放哪 | API 从哪来 | 要不要 Docker |
|------------|----------|------------|----------------|
| 先上线、用别人 API | Vercel | 填官方或别人给的 API 地址 | **不需要** |
| 自建 API、完整自己控 | Vercel | 自己服务器跑 API（Docker 或 Node） | **只给 API 用**，前端不用 Docker |

- **Vercel**：只负责「静态页面」，和「有没有 Cobalt」不冲突；有没有 Cobalt 取决于你**填的 API 地址**是谁在跑。  
- **Docker**：只用来在服务器上跑 **Cobalt API**，不用把前端塞进 Docker。  
- 前端构建产物就是 `web/build` 里那堆静态文件，任何能托管静态站的地方（Vercel / Netlify / 自己的 Nginx）都能用。

按上面步骤：先 Vercel 部署前端并设好 **WEB_DEFAULT_API**，若选自建再按「四」在服务器起 API 即可。
